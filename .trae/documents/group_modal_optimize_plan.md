# 编辑组弹窗优化计划

## 目标
将编辑组弹窗改造成基于Tab切换的UI，权限管理和成员管理使用统一的双列表布局样式。

## 主要修改内容

### 1. 更新 GroupEditModal.tsx
**修改内容：**
- 添加tab切换功能，包含三个Tab：
  - 基本信息
  - 权限管理
  - 成员管理
- 保持权限管理的双列表UI
- 将成员管理也改成双列表UI：
  - 左侧：当前组成员（点击可移除）
  - 右侧：其他组成员（点击可添加到当前组）
- 调用API时，成员修改使用 `/user group` 命令或 changeGroup API

### 2. 可能需要更新的其他组件
- 确保 ServerStatusView.tsx 传入正确的 props
- 如有需要，确保 useTShock.ts 中的函数正确暴露

## 实现步骤
1. 添加activeTab状态来管理当前tab
2. 创建tab按钮组件，支持切换
3. 重构基本信息区域到tab中
4. 重构权限管理区域到tab中（保持现有样式）
5. 重构成员管理为双列表UI
6. 添加成员添加/移除的处理函数
7. 调整整体布局

## 技术要点
- 使用现有的 changeGroup API 来修改用户组
- 成员管理的双列表UI设计要与权限管理一致
- 保持响应式设计
