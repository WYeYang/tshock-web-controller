# 解封后封禁列表数量没减少问题分析

## 问题描述
用户点击解封按钮后，收到解封成功的提示，但封禁列表的数量没有减少。

## 可能的原因分析

### 1. 解封 API 调用问题
**可能原因**：解封 API 使用了错误的 HTTP 方法

**分析**：
- 当前代码使用 `GET` 方法调用 `/v3/bans/destroy`
- 但根据 RESTful 规范，删除操作通常应该使用 `POST` 或 `DELETE` 方法
- 虽然 TShock 文档没有明确指定方法，但很多 RESTful API 在删除操作时需要使用 POST

**验证**：需要测试确认 TShock API 支持的 HTTP 方法

### 2. 解封成功后数据没有刷新
**可能原因**：解封操作看起来成功了，但实际上 API 调用失败或返回错误

**分析**：
- `confirmAction` 函数中解封成功后调用了 `fetchData()`
- 但如果 `fetchData()` 中的 `getBanList()` 失败，可能导致状态没有更新
- 或者 UI 没有正确响应状态变化

**验证**：
- 检查浏览器控制台是否有错误
- 检查 network 面板中的 API 响应

### 3. 数据类型不匹配
**可能原因**：`ticket_number` 类型问题

**分析**：
- API 返回的 `ticket_number` 是数字类型
- 但在 `unbanPlayer` 调用时可能传递了字符串类型
- 导致 API 无法正确识别要删除的记录

**验证**：检查 `unbanPlayer` 调用时传递的参数类型

### 4. 解封只是标记为过期而非删除
**可能原因**：解封操作没有真正删除记录

**分析**：
- 有些封禁系统的"解封"只是修改封禁状态为"已过期"
- 而不是从数据库中真正删除记录
- 因此 `getBanList()` 仍然返回所有记录

**验证**：需要查看 API 返回的 `end_date_ticks` 字段来判断

## 解决方案

### 方案 1：尝试使用 POST 方法
如果 GET 方法不行，尝试修改 `unbanPlayer` 函数使用 POST：

```typescript
async unbanPlayer(ticketNumber: number, fullDelete: boolean = false): Promise<any> {
  let endpoint = `/v3/bans/destroy`;
  const body = new URLSearchParams({
    ticketNumber: ticketNumber.toString(),
    ...(fullDelete && { fullDelete: 'true' }),
  });
  
  return this.request<any>(endpoint, {
    method: 'POST',
    body,
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded',
    },
  });
}
```

### 方案 2：添加错误处理和重试机制
确保 API 调用失败时能够捕获并报告错误

### 方案 3：使用命令方式解封
如果 API 不稳定，可以改用执行 `/ban del <ticketNumber>` 命令

## 实施步骤

1. **立即修复**：将解封 API 从 GET 改为 POST 方法
2. **添加调试**：在关键位置添加 console.log
3. **测试验证**：重新测试解封功能
4. **如果还不行**：改用命令方式 `/ban del <ticketNumber>`
