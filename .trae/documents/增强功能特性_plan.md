# 功能特性增强方案

## 需求分析

在下载页面的"功能特性"部分，加入TShock的特色功能描述，突出与原版泰拉瑞亚服务器的差异，展示各种特色指令和管理功能。
- TShock 服务器核心特色放前面
- 加入强制开荒功能（服务端角色 SSC）
- 随机添加一些物品图标作为装饰

## 当前状态

当前功能特性包括：
1. 服务器状态监控
2. 用户和用户组管理
3. 玩家信息查看
4. 配置面板
5. 终端面板
6. 内置 TShock 服务器

## 改进方案

### 方案概述

将功能特性分为两个部分：
1. **TShock 服务器核心特色** - TShock 本身提供的强大功能（放前面）
2. **TShock 控制器特色功能** - 本应用提供的管理功能

### 详细设计

#### 第一部分：TShock 服务器核心特色（9 个特性）

| 功能 | 描述 | 代表性命令 |
|------|------|-----------|
| 强制开荒（SSC） | 服务端角色系统，强制玩家从零开始，背包和角色数据保存于服务器 | `ServerSideCharacter` |
| 玩家管理 | 踢出、封禁、禁言玩家，维护服务器秩序 | `/kick`, `/ban`, `/mute` |
| 物品管理 | 给予玩家物品，管理物品禁令，防止作弊 | `/give`, `/item`, `/itemban` |
| 传送系统 | 丰富的传送功能，包括玩家间传送、传送点、坐标传送 | `/tp`, `/warp`, `/tppos` |
| 世界事件控制 | 手动触发血月、日食、入侵等事件，生成Boss和怪物 | `/worldevent`, `/spawnboss`, `/butcher` |
| 世界管理 | 控制时间、天气、难度，设置出生点和地牢位置 | `/time`, `/rain`, `/hardmode` |
| 区域保护 | 创建受保护区域，防止恶意破坏 | `/region` |
| 玩家功能 | 治愈、增益、上帝模式、复活等管理员功能 | `/heal`, `/buff`, `/godmode` |
| 聊天系统 | 队伍聊天、私聊、广播消息等完善的聊天功能 | `/party`, `/whisper`, `/broadcast` |

#### 第二部分：TShock 控制器特色功能（6 个特性）

| 功能 | 描述 |
|------|------|
| 服务器状态监控 | 实时查看服务器运行状态、玩家在线情况和性能指标 |
| 用户和用户组管理 | 可视化界面管理用户权限和用户组配置 |
| 玩家信息查看 | 查看玩家详细信息、背包、装备和属性 |
| 配置面板 | 图形化配置服务器参数，无需手动编辑配置文件 |
| 终端面板 | 内置终端支持直接执行服务器命令 |
| 内置 TShock 服务器 | 桌面版包含完整的 TShock 服务器，开箱即用 |

### 物品图标装饰

在页面合适位置添加一些泰拉瑞亚物品图标作为装饰元素：

1. **页面顶部装饰**：在标题下方添加一排随机物品图标
2. **功能卡片装饰**：在每个功能卡片旁边或内部添加相关主题的物品图标
3. **底部装饰**：在页面底部添加一排物品图标

部分代表性物品：
- 天顶剑 (Zenith) - 最强武器
- 泰拉瑞亚 (Terrarian) - 最终Boss掉落
- 各种矿石、工具、武器、配饰

### 页面布局设计

1. **保持现有 2 列网格布局**
2. **第一部分**：TShock 服务器核心特色（9 个卡片）
3. **第二部分**：TShock 控制器特色功能（6 个卡片）
4. **每个卡片包含**：
   - 图标
   - 标题
   - 描述
   - 代表性命令（仅第一部分显示）
5. **物品图标装饰**：在页面适当位置随机分布物品图标

### 实施步骤

1. 修改 `DownloadPage.tsx` 的功能特性数据结构
2. 将功能特性分为两个分组
3. 优化每个特性的描述，加入更多细节
4. 为 TShock 核心特色添加代表性命令展示
5. 导入 `getItemIconUrl` 和相关类型
6. 添加装饰性物品图标组件
7. 在页面合适位置放置物品图标
8. 保持现有的视觉风格和动画效果
