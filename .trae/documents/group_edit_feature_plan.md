
# 用户组编辑功能改造计划

## 概述
本次改造旨在改进用户组管理功能，将现有的卡片展开编辑模式改为弹窗编辑模式，并优化权限选择体验。

## 需求分析

### 1. 移除卡片展开功能
- 去除用户组列表中的卡片展开/收起按钮
- 简化用户组列表展示，只显示基本信息
- 移除 `expandedGroup` 相关状态和逻辑

### 2. 添加弹窗编辑模式
- 创建独立的编辑用户组弹窗组件
- 点击"编辑"按钮时打开弹窗
- 弹窗包含：用户组基本信息编辑、权限选择区域

### 3. 权限编辑改为左右双列表
- 左侧：可用权限列表（按分类显示）
- 右侧：已选权限列表
- 点击权限项即可在两列表之间移动
- 支持全选/全取消功能

### 4. 每次移动同步请求API
- 当用户添加/移除权限时，立即调用API更新
- 提供实时反馈（成功/失败提示）
- 保持状态与服务器同步

## 实现方案

### 主要修改文件
`src/components/ServerStatusView.tsx`

### 新增状态变量
```typescript
- editGroupDialogOpen: boolean  // 编辑弹窗是否打开
- permissionLoading: boolean     // 权限操作加载状态
```

### 新增/修改组件
1. **EditGroupDialogComp** - 编辑用户组弹窗组件
2. **PermissionDualList** - 双列权限选择组件（内嵌在弹窗中）

### 权限列表结构
- 保持使用 `CATEGORIZED_PERMISSIONS` 常量
- 左侧：未选中的权限
- 右侧：已选中的权限

### API调用逻辑
- 修改 `updateExistingGroup` 的使用方式
- 添加单权限更新的便捷函数
- 每次权限变更时调用API

## 详细步骤

### 步骤1：移除卡片展开功能
1. 移除 `expandedGroup` 状态
2. 移除 `toggleGroupExpansion` 函数
3. 简化用户组列表渲染，去除展开/收起区域
4. 调整编辑按钮逻辑，直接打开弹窗

### 步骤2：添加弹窗编辑模式
1. 添加 `editGroupDialogOpen` 状态
2. 修改 `startEditGroup` 函数，打开弹窗而非展开卡片
3. 创建 `EditGroupDialogComp` 组件
4. 将原编辑表单迁移到弹窗中

### 步骤3：实现双列权限选择
1. 将权限列表分为两列布局
2. 实现权限点击移动功能
3. 添加全选/取消全选按钮
4. 保留按分类显示的结构

### 步骤4：实现实时API同步
1. 创建 `handlePermissionChange` 函数
2. 在权限变更时调用 `updateExistingGroup`
3. 添加加载状态和错误处理
4. 优化用户体验反馈

### 步骤5：优化UI和UX
1. 调整样式适配新布局
2. 添加加载指示器
3. 优化提示消息
4. 确保响应式设计

## 技术要点

### 状态管理
- 使用本地状态管理弹窗和权限选择
- 每次操作后重新获取最新用户组数据

### API调用
- 使用现有的 `updateExistingGroup` 函数
- 每次权限变更发送完整权限列表

### 错误处理
- 保持现有的 toast 提示机制
- 添加操作失败时的回滚逻辑

## 预期效果

1. **更简洁的用户组列表** - 去除展开功能，界面更清爽
2. **更直观的编辑体验** - 弹窗模式更符合现代UI设计
3. **更高效的权限管理** - 双列点击移动，操作更便捷
4. **更可靠的数据同步** - 实时更新，避免数据不一致

## 风险评估

- **API调用频率** - 频繁调用可能影响性能，但用户组编辑频率较低，可接受
- **权限列表过长** - 分类显示和搜索（可选）可缓解此问题
- **数据一致性** - 本地状态与服务器同步需仔细处理
