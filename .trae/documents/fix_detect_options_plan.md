# 修复选项检测问题

## 问题分析

`detectOptions` 函数只检测带方括号的格式（如 `[4]`），但TShock显示的世界列表是纯数字格式：

```
泰拉瑞亚服务器v1.4.5.6
1		可恨的小猫源头
2		神话酒馆
3		天顶世界
4		天顶世界-极限空岛
n		新建世界
d <number>	删除世界
选择世界：
```

导致没有显示可点击的选项按钮，用户只能手动输入。

## 修改计划

只修改 `src/components/SetupWizard.tsx`：

1. 更新 `detectOptions` 函数，检测：
   - 带方括号的格式 `[4]`
   - 不带方括号的纯数字 `1`、`2`、`3`、`4`
   - 特殊选项 `n`、`d`

2. 检测"选择世界"等提示文本

## 实施步骤

1. 修改 `detectOptions` 函数
2. 测试是否正确显示选项按钮

## 预期结果

- 世界列表中的1、2、3、4、n、d都显示为可点击按钮
- 用户点击按钮就能选择，不需要手动输入
