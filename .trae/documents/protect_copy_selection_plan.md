# 保护终端选中复制功能计划

## 问题分析
当前终端虽然禁用了输入，但用户担心终端启动的程序可能会影响选中复制功能。需要确保：
1. 终端完全禁止任何输入
2. 保持完整的文本选择和复制功能
3. 防止终端程序修改选择行为

## 研究结论
xterm.js 提供了多个配置选项来控制终端行为，我们可以通过配置确保：
- `disableStdin: true` - 禁用标准输入
- `allowProposedApi: false` - 禁用可能影响选择的实验性 API
- 通过事件监听确保选择功能正常

## 需要修改的文件
1. `src/components/TerminalUI.tsx` - 终端组件
2. 可能需要查看 xterm.js 文档确认最佳实践

## 修改步骤

### 步骤 1: 增强终端配置
在 TerminalUI.tsx 中添加更多 xterm.js 配置选项：
- `cursorInactiveStyle: 'none'` - 隐藏光标
- `allowProposedApi: false` - 禁用实验性 API
- `convertEol: true` - 保持不变
- `scrollback: 5000` - 保持不变

### 步骤 2: 确保选择功能独立
- 改进 `handleSelectionChange` 函数，确保只在终端区域内的选择才触发
- 添加 `mousemove` 事件监听来更新选择状态
- 确保复制功能使用终端自己的 `getSelection()` 方法

### 步骤 3: 阻止所有可能干扰选择的事件
- 完全禁用 textarea 的所有交互
- 添加更多事件监听来确保选择不被干扰

### 步骤 4: 测试验证
确保：
- 可以正常选择文本
- 点击"复制"按钮可以正常复制
- 终端输出不会干扰选择状态

## 依赖和注意事项
- xterm.js 版本兼容性
- 不同浏览器的选择行为可能有差异
- 需要确保不影响终端的正常输出显示

## 风险处理
- 如果某个配置导致问题，可以回滚到之前的状态
- 保持简单，不要过度复杂化
