# 配置弹窗添加 world 文件配置

## 摘要
在 TShock 配置弹窗中添加以下文本文件的编辑功能：
- sscconfig.json（与 config.json 逻辑相同，仅路径不同）
- motd.txt
- rules.txt
- whitelist.txt

## 当前状态分析

### 后端架构
- **electron/ipc/config.js**：提供 config.json 的读取/写入
- **electron/preload.cjs**：暴露 IPC handlers 给前端
- **src/services/electronBridge.ts**：前端访问后端 API 的桥接层

### 可复用部分
- `readConfig(filePath)` 和 `writeConfig(filePath, data)` 已经可以接受任意文件路径
- 只需修改 IPC handlers 来接受文件名参数

### 文件路径结构
```
TShock/tshock/
├── config.json       # 已有
├── sscconfig.json    # 需要添加
├── motd.txt          # 需要添加
├── rules.txt         # 需要添加
└── whitelist.txt     # 需要添加
```

## 实施计划

### 1. 修改 electron/ipc/config.js

修改现有 IPC handlers：
- `config:read` 接受 `filename` 参数（可选，默认 config.json）
- `config:write` 接受 `filename` 和 `data` 参数（filename 可选，默认 config.json）
- 路径直接通过 `path.join(getTShockRootDir(), 'tshock', filename)` 拼接

### 2. 修改 electron/preload.cjs

更新 IPC 绑定以支持参数

### 3. 修改 src/services/electronBridge.ts

更新 config 接口以接受文件名参数

### 4. 修改 src/components/WizardConfigEditorModal.tsx

在配置弹窗中添加：
1. **sscconfig.json** - 使用 JSON 编辑器
2. **motd.txt** - 多行文本输入框（textarea）
3. **rules.txt** - 多行文本输入框（textarea）
4. **whitelist.txt** - 多行文本输入框（textarea）

加载时读取所有文件，保存时写入所有文件。
