# 自动检测代理方案分析

## 问题背景

当前 Web 版和桌面版的 REST 请求地址处理方式不一致：
- 桌面版（Electron）：直接请求完整 URL（`http://localhost:7878/v2/server/status`）
- Web 版：请求相对路径（`/api/v2/server/status`）由代理转发

## 解决方案：智能检测 + 自动降级

### 核心思路

1. **Web 版优先尝试直接请求完整 URL**（与桌面版一致）
2. **自动检测是否成功**
   - 成功：继续使用直接请求
   - 失败（特别是 CORS 错误）：自动降级到代理路径
3. **提供友好的用户提示**

### 实现细节

#### 1. 检测机制

在 `tshockApi.ts` 中添加检测方法：
- 尝试发送一个简单的直接请求（如 `/v2/server/status`）到完整 URL
- 监听请求错误
- 区分 CORS 错误和其他错误

#### 2. 降级策略

- 建立缓存机制，记录检测结果（避免每次都检测）
- 检测失败后，自动切换到代理路径模式
- 提供用户手动切换的选项

#### 3. 用户提示

在 UI 中添加提示：
- 直接请求成功：静默处理
- 直接请求失败：显示提示，告诉用户可以配置 TShock CORS 或使用代理

### 修改文件

- [tshockApi.ts](file:///workspace/src/services/tshockApi.ts)：核心逻辑修改
- 相关 UI 组件：添加提示信息（可选）

### 优势

- ✅ 代码逻辑统一，Web 版和桌面版尽可能一致
- ✅ 用户体验优化，自动处理跨域问题
- ✅ 保留两种方式的选择权
