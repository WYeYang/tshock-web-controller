# 用户组管理功能 - 产品需求文档

## 概述
- **摘要**: 在 TShock Web 控制器的服务器状态页面添加用户组管理功能，允许管理员查看、创建、编辑和删除用户组，以及管理组成员和权限。
- **目的**: 提供一个直观的 Web 界面来管理 TShock 服务器的用户组和权限系统，无需通过游戏内命令或直接编辑配置文件。
- **目标用户**: TShock 服务器管理员

## 目标
- 在服务器状态页面添加「用户组」tab
- 支持查看用户组列表和详情
- 支持创建、编辑、删除用户组
- 支持编辑用户组权限（复选框形式）
- 支持查看和管理用户组成员

## 非目标（超出范围）
- 不实现权限的层级继承可视化
- 不实现批量权限编辑
- 不实现用户组导入/导出功能

## 背景与上下文
现有 TShock Web 控制器已经支持：
- 服务器状态监控
- 在线玩家管理（踢出、封禁、禁言、修改用户组）
- 封禁列表管理
- 通过 API 调用执行命令

需要新增的功能是用户组管理，这是 TShock 权限系统的核心部分。

## 功能需求
- **FR-1**: 在服务器状态页面添加「用户组」tab
- **FR-2**: 显示用户组列表（组名、父组、权限数、成员数）
- **FR-3**: 查看用户组详情（展开卡片形式）
- **FR-4**: 创建新用户组
- **FR-5**: 编辑用户组（基本信息、权限）
- **FR-6**: 删除用户组
- **FR-7**: 查看用户组成员列表
- **FR-8**: 修改用户的用户组（成员管理）

## 非功能需求
- **NFR-1**: 界面风格与现有 UI 保持一致
- **NFR-2**: 操作有明确的成功/失败反馈
- **NFR-3**: 加载状态有适当的视觉提示

## 约束
- **技术**: 使用现有的 React + TypeScript + Tailwind CSS 技术栈
- **依赖**: TShock REST API 的用户组相关端点
- **权限**: 需要 `tshock.rest.groups.manage` 和 `tshock.rest.groups.view` 权限

## 假设
- TShock REST API 正常工作且配置正确
- 用户已有访问用户组管理 API 的权限
- 权限列表使用预定义的常用权限集合

## 验收标准

### AC-1: 用户组 Tab 显示
- **前置条件**: 已登录并进入服务器状态页面
- **操作**: 点击「用户组」tab
- **结果**: 显示用户组列表，每个组显示名称、父组、权限数、成员数
- **验证类型**: human-judgment
- **备注**: 风格与现有玩家/封禁列表一致

### AC-2: 查看用户组详情
- **前置条件**: 用户组列表已加载
- **操作**: 点击某个用户组卡片
- **结果**: 卡片展开，显示该组的详细信息（基本信息、权限列表、成员列表）
- **验证类型**: human-judgment

### AC-3: 创建用户组
- **前置条件**: 用户组列表已加载
- **操作**: 点击「+ 创建新组」，填写信息并确认
- **结果**: 新用户组创建成功并显示在列表中
- **验证类型**: human-judgment

### AC-4: 编辑用户组权限
- **前置条件**: 用户组详情已展开
- **操作**: 在权限列表中勾选/取消勾选权限并保存
- **结果**: 权限更新成功
- **验证类型**: human-judgment

### AC-5: 删除用户组
- **前置条件**: 用户组详情已展开
- **操作**: 点击删除按钮并确认
- **结果**: 用户组被删除，从列表中移除
- **验证类型**: human-judgment

### AC-6: 成员管理
- **前置条件**: 用户组详情已展开
- **操作**: 查看成员列表，或修改某个用户的组
- **结果**: 可以查看成员，并且可以成功修改用户的组
- **验证类型**: human-judgment
