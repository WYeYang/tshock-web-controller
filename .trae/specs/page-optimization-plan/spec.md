# TShock 控制器页面优化 - 产品需求文档

## 概述

- **总结**：优化 TShock 控制器的页面结构，删除不需要的页面，合并功能页面，改进配置体验
- **目的**：简化用户界面，提供更紧凑、更易用的服务器管理体验
- **目标用户**：TShock 服务器管理员

## 目标

- 删除自然语言、历史、用户页面
- 将状态、命令、帮助合并到一个主页面
- 将配置页面改造成顶部可折叠面板
- 提供更流畅的用户体验

## 非目标（排除范围）

- 不改变核心功能逻辑
- 不改变 API 调用方式
- 不添加新的功能

## 背景与上下文

当前项目有 7 个页面：
- ConfigPage - 配置页面
- StatusPage - 状态页面
- CommandPage - 命令页面
- NLPPage - 自然语言页面
- HistoryPage - 历史页面
- UserPage - 用户页面
- HelpPage - 帮助页面

## 功能需求

### FR-1: 删除不需要的页面

删除 NLPPage、HistoryPage、UserPage 及其相关文件

### FR-2: 创建新的主页面 Dashboard

整合 StatusPage、CommandPage、HelpPage 的功能到一个页面，通过选项卡或分段组织内容

### FR-3: 改造配置页面为可折叠面板

将 ConfigPage 改造成顶部可折叠的配置卡片组件，集成到主页面顶部

### FR-4: 更新导航和路由

更新 Navbar 和 PageRouter 来适应新的页面结构

### FR-5: 配置卡片智能展开/收起

- 无配置时默认展开
- 有配置时默认收起
- 用户可以手动切换

## 非功能需求

- 保持现有代码的功能完整性
- 保持一致的视觉风格
- 确保平滑的过渡体验

## 约束条件

- **技术约束**：基于现有 React + TypeScript 代码库
- **依赖约束**：使用现有组件和状态管理
- **设计约束**：保持原有的 UI 风格

## 假设条件

- 用户希望保留所有现有功能，只是重新组织布局
- 配置管理仍然是独立的重要功能

## 验收标准

### AC-1: 页面删除

- **给定**：项目构建成功
- **当**：检查代码库
- **则**：NLPPage、HistoryPage、UserPage 文件已删除，导航中不再显示这些选项
- **验证方式**：programmatic

### AC-2: 新主页面集成

- **给定**：访问应用首页
- **当**：查看主页面
- **则**：可以看到服务器状态、命令控制台、帮助文档三个功能区块（或选项卡）
- **验证方式**：human-judgment

### AC-3: 配置折叠面板

- **给定**：第一次访问应用（无配置）
- **当**：查看页面顶部
- **则**：配置面板默认展开
- **验证方式**：human-judgment

### AC-4: 配置面板智能收起

- **给定**：已保存配置并刷新页面
- **当**：查看页面顶部
- **则**：配置面板默认收起，点击可展开
- **验证方式**：human-judgment

### AC-5: 功能完整性

- **给定**：配置已正确设置
- **当**：使用各项功能
- **则**：所有原有功能（踢人、封禁、执行命令等）正常工作
- **验证方式**：programmatic + human-judgment
